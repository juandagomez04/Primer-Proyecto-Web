# Primer-Proyecto-Web: Aventones

Este proyecto es un prototipo funcional de una aplicación web tipo "cascarón", desarrollada como parte del curso **Diseño de Aplicaciones Web (ISW-512)** en la carrera de Ingeniería del Software de la Universidad Técnica Nacional.

## 🎯 Objetivo

Simular una plataforma de viajes compartidos ("carpooling") donde los usuarios pueden registrarse como clientes o proveedores, y realizar acciones relacionadas con la creación, edición y búsqueda de viajes (rides), sin almacenamiento persistente de datos.

## 🧩 Funcionalidades principales

- ✅ **Registro de usuario cliente** con datos básicos como nombre, cédula, correo, teléfono, etc.
- ✅ **Registro de usuario proveedor** incluyendo datos del vehículo.
- ✅ **Inicio de sesión** simulado (formulario y opción Google).
- ✅ **Gestión de viajes (Rides)**: crear, editar, eliminar y ver detalle.
- ✅ **Pantalla My Rides** para proveedores autenticados.
- ✅ **Búsqueda pública de viajes** según origen y destino.
- ✅ **Formulario para editar perfil del usuario.**
- ✅ **Pantalla de Bookings** para ver el estado de solicitudes de viaje.

## 🛠️ Tecnologías utilizadas

- HTML5
- CSS3 (sin frameworks, uso de variables CSS)
- Git (buenas prácticas de commits)
- Validación W3C
- XAMP (Servidor local APACHE)

## 📁 Estructura del proyecto

- Archivos `.html` para cada pantalla del sistema
- Archivos `.css` individuales por contexto (home, rides, bookings, etc.)
- Carpeta `Images/` con recursos gráficos

## 🧪 Consideraciones

- Toda la lógica de interacción es simulada desde el frontend.
- Navegación entre páginas mediante `href`.
